Laboratory Work

Mobile Dev Labs

Lab 1

User Interface Design

Designed and built foundational Android UI layouts using Views, ConstraintLayout, and XML to establish a clean and consistent screen structure.

This lab taught me the importance of visual hierarchy and responsive layout design. Mastering ConstraintLayout gave me the confidence to structure complex screens without relying on nested views. I now understand how good UI design forms the backbone of a usable application.
Lab 2

Interaction & Event Handling

Implemented button listeners, input validation, and inter-activity navigation to create responsive user interactions in Kotlin.

Handling user interactions made the app feel truly alive. I learned how intents and callbacks tie the UI layer to business logic cleanly. Debugging event chains helped me develop a more systematic approach to problem solving in Android development.
Lab 3

Material Design

Applied Material Design 3 principles — theming, typography, colour systems, and components like FABs, cards, and dialogs — to produce polished UIs.

Material Design gave me a shared design language that bridges aesthetics and functionality. Using Material components saved development time while maintaining accessibility standards. I gained a deeper appreciation for how design systems improve team productivity and product consistency.
Lab 4

Navigation Component

Implemented Jetpack Navigation with a NavHost, fragments, and a bottom navigation bar to manage multi-screen app flow efficiently.

The Navigation Component dramatically simplified back-stack management and deep-link handling. Separating navigation logic from fragment code made the project far easier to maintain. I now understand why single-activity architecture is the modern best practice for Android apps.
Lab 5

Room Persistence Library

Integrated Room with DAO interfaces, entities, and LiveData observers to provide reliable offline-first local data persistence.

Room abstracted away the complexity of raw SQLite while enforcing safe data access patterns at compile time. Working with LiveData and the Observer pattern solidified my understanding of reactive programming. Offline-first design has become a core principle I carry into every project.

Projects

Project 1 · Lab 4 Integration

NextStop

Problem Statement

Urban commuters in Malaysian cities struggle to track public transit routes and arrival times efficiently, leading to missed connections and wasted time. NextStop solves this by providing real-time route guidance directly on a user's smartphone.

Building NextStop pushed me to combine navigation architecture with real feature implementation for the first time. Managing state across multiple fragments and keeping the UI responsive taught me practical MVVM patterns. I also learned the value of user testing early — feedback revealed navigation flows I had overlooked.

Project 2 · Lab 5 + API + Sensors + Cloud

SmartCity Pulse

Problem Statement

City residents lack a unified view of local environmental data — air quality, noise levels, and transit crowding — making it hard to make informed daily decisions. This app aggregates sensor data and public APIs into a single real-time dashboard.

Integrating Retrofit for API calls alongside CameraX for sensor input required careful asynchronous flow management with Kotlin Coroutines. Syncing local Room data with Firebase Firestore introduced me to conflict-resolution strategies in offline-first architectures. This project was the most technically demanding of the semester and gave me the greatest confidence boost as a developer.

Learning Journey

Reflection

This semester has been a transformative journey from writing my first line of Kotlin to architecting a cloud-connected, sensor-driven mobile application. I entered with a basic understanding of programming and leave with hands-on experience in modern Android development — from Material Design and Jetpack Navigation all the way to Room databases and Firebase integration.


The SDG 11 theme — Sustainable Cities and Communities — gave my work a clear sense of purpose. Every lab and project felt grounded in a real-world challenge, which pushed me to think beyond just making code run and instead consider who would use it and why it matters.


The biggest lesson I carry forward is that software is never truly finished — it evolves through iteration, user feedback, and a willingness to refactor. Under Sir Izwan's guidance, I learned not just technical skills but also how to think critically as a developer and communicate my design decisions with confidence.


I look forward to deepening my expertise in Jetpack Compose and cloud-native architectures, and continuing to build applications that contribute positively to the communities around me.
